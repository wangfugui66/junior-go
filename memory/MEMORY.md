# MEMORY.md — curated, durable project knowledge

This file is the small, high-signal layer of `memory/`: short one-line facts distilled from
`memory/notes/*.md` by the human orchestrator at a PASS acceptance checkpoint (see
`memory/README.md`). Read it whole, every retrieval — it's meant to stay small enough that doing so
is cheap. Add a line here only when a human has actually promoted it; don't append directly from an
agent run. Full protocol, schema, and the promotion/cleanup rules: `memory/README.md`.

**This particular copy, in this harness repo, is a schema example — see `memory/README.md`'s
"Where this lives at runtime" section. It is not read by any agent at runtime against a real
target project; a real project gets its own `memory/MEMORY.md` at its own root.**
